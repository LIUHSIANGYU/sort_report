## 氣泡排序法

### 原理
氣泡排序是最簡單的排序演算法。它通過不斷比較相鄰的元素，將較大的元素交換到右邊，使得大的數值像氣泡一樣逐步「浮起」到陣列的右端。每次遍歷都會將最大的元素移到末尾。

### 複雜度分析
- **時間複雜度**：
  - 最壞情況：O(n²) - 當陣列完全反序排列時，需要最多的比較和交換
  - 平均情況：O(n²) - 隨機排列的陣列需要約 n²/4 次比較
  - 最好情況：O(n) - 當陣列已排序時，只需 n-1 次比較
- **空間複雜度**：O(1) - 只需常數空間，屬於 In-place 排序

### 模擬實作內容
互動演示頁面提供：
- 輸入自定義陣列或生成隨機陣列
- 視覺化顯示排序過程中的比較和交換
- 即時統計比較次數和交換次數
- 可調整排序速度以觀察過程
- 顯示排序前後的陣列狀態

## 選擇排序法

### 原理
選擇排序是一種簡單的排序法。每次從未排序的元素中找到最小值，放在已排序部分的末尾。通過這樣的選擇過程，整個陣列逐步建立了有序序列。

### 複雜度分析
- **時間複雜度**：
  - 最壞情況：O(n²) - 無論陣列排列如何，都需要完整的比較
  - 平均情況：O(n²) - 隨機排列的陣列平均需要約 n²/2 次比較
  - 最好情況：O(n²) - 即使陣列已排序，仍需進行完整的 n²/2 次比較
- **空間複雜度**：O(1) - 只需常數空間，屬於 In-place 排序

- 即時統計比較次數和交換次數
- 可調整排序速度以觀察過程
- 顯示排序前後的陣列狀態

## 插入排序法

### 原理
插入排序透過逐步建立有序序列的方式進行排序。它將陣列分為已排序和未排序兩部分，每次從未排序部分取出一個元素，將其插入到已排序部分的正確位置。

### 複雜度分析
- **時間複雜度**：
  - 最壞情況：O(n²) - 當陣列完全反序排列時，需要最多的比較
  - 平均情況：O(n²) - 隨機排列的陣列需要約 n²/4 次比較
  - 最好情況：O(n) - 當陣列已排序時，只需 n-1 次比較
- **空間複雜度**：O(1) - 只需常數空間，屬於 In-place 排序

- 可調整排序速度以觀察過程

## 合併排序法

### 原理
合併排序是一種分治演算法，採用「分而治之」的策略。它將陣列分割成更小的子陣列，然後遞迴地對每個子陣列進行排序，最後將已排序的子陣列合併回一個完整的排序陣列。

### 複雜度分析
- **時間複雜度**：
  - 最壞情況：O(n log n) - 無論陣列如何排列，複雜度始終是 n log n
  - 平均情況：O(n log n) - 隨機排列的陣列也需要 n log n 次操作
  - 最好情況：O(n log n) - 即便陣列已排序，仍需 n log n 次操作
- **空間複雜度**：O(n) - 需要額外空間存儲臨時陣列

### 模擬實作內容

- 輸入自定義陣列或生成隨機陣列
- 視覺化顯示分割和合併的過程
- 即時統計比較次數和合併次數
- 可調整排序速度以觀察過程
- 顯示排序前後的陣列狀態

## 快速排序法

### 原理
快速排序是一種分散-征服演算法，它通過選擇一個樞紐元素（pivot），將陣列分為兩部分：小於樞紐的元素和大於樞紐的元素。然後對這兩部分遞迴地進行相同的過程，直到整個陣列排序完成。

### 複雜度分析
- **時間複雜度**：
  - 最壞情況：O(n²) - 當樞紐始終為最小或最大元素時，每次只分割出 1 個元素
  - 平均情況：O(n log n) - 樞紐較為平衡時，陣列平均被分成兩個相等的部分
  - 最好情況：O(n log n) - 樞紐始終為中間值時
- **空間複雜度**：O(log n) - 遞歸調用棧的空間

## 綜合比較

| 排序演算法 | 時間複雜度 (平均) | 空間複雜度 | 穩定性 | 適用場景 |
|-----------|------------------|-----------|-------|---------|
| 氣泡排序 | O(n²) | O(1) | 穩定 | 小規模資料，教學演示 |
| 選擇排序 | O(n²) | O(1) | 不穩定 | 小規模資料，簡單實現 |
| 插入排序 | O(n²) | O(1) | 穩定 | 部分有序資料，線上排序 |
| 合併排序 | O(n log n) | O(n) | 穩定 | 大規模資料，外部排序 |
| 快速排序 | O(n log n) | O(log n) | 不穩定 | 大規模資料，通用排序 |

### 比較分析
- **時間效率**：合併排序和快速排序在平均情況下最快，適合大規模資料；氣泡、選擇、插入排序適合小規模資料。
- **空間效率**：氣泡、選擇、插入排序為 In-place 排序，空間效率最高；合併排序需要額外空間；快速排序空間效率中等。
- **穩定性**：氣泡、插入、合併排序為穩定排序，選擇和快速排序不穩定。
- **實用性**：快速排序因平均性能優秀且空間效率較好，在實務中最常用；合併排序適合需要穩定排序的場合。

## 學習心得

在這次排序演算法的學習與專題實作中，我從只會照著範例打程式，進一步學會分析不同演算法的特性與適用情境。我實作了氣泡排序、插入排序、選擇排序、合併排序與快速排序，實際觀察它們在不同資料量與不同資料分布下的表現。當資料只有幾十筆時，大部分演算法的速度看起來差不多；但當資料增加到幾百甚至幾千筆時，那些需要一一比較、反覆掃描整個陣列的演算法會明顯變慢，而設計較佳的演算法則可以在較短時間內完成排序，讓我實際感受到「演算法效率好不好」在實務上的差異。

在專題中，我使用 HTML 製作互動式排序演示頁面，將原本抽象的演算法步驟變成長條圖動畫。使用者可以選擇演算法、調整資料數量與播放速度，並透過播放、重播按鈕一步步觀察比較與交換的過程。

過程中我也遇到不少問題，例如快速排序遞迴終止條件寫錯導致程式無限呼叫自己，或是因為忘記複製陣列而影響多種排序的比較結果。透過除錯工具、輸出中間狀態以及把程式模組化，我學會更有系統地定位問題並修正。整體來說，這次專題讓我真正體會到，演算法不只是紙上的流程，而是會直接影響程式在真實資料上的速度與使用體驗。未來我希望能在這次經驗的基礎上，繼續學習更多演算法與資料結構，並應用在更大型或更實務的專案中。
