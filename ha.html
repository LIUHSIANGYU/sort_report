氣泡排序法的原理

氣泡排序是最簡單的排序演算法。它通過不斷比較相鄰的元素，將較大的元素交換到右邊，使得大的數值像氣泡一樣逐步「浮起」到陣列的右端。

基本步驟：

  1. 比較相鄰的兩個元素
  2. 如果前一個元素大於後一個元素，則交換它們
  3. 移動到下一對元素，重複步驟 1-2
  4. 每次完整遍歷後，最大的元素會移到末尾
  5. 重複此過程，直到陣列完全排序

JavaScript 實現：

function bubbleSort(arr) { const n = arr.length; let comparisons = 0; let swaps = 0; for (let i = 0; i < n - 1; i++) { let swapped = false; for (let j = 0; j < n - i - 1; j++) { comparisons++; if (arr[j] > arr[j + 1]) { // 交換元素 [arr[j], arr[j + 1]] = [arr[j + 1], arr[j]]; swaps++; swapped = true; } } // 優化：若沒有交換則已排序 if (!swapped) break; } return { comparisons, swaps }; }

⏱️ 時間與空間複雜度

最壞情況

O(n²)

陣列完全反序排列時需要最多的比較和交換

平均情況

O(n²)

隨機排列的陣列需要約 n²/4 次比較

最好情況

O(n)

陣列已排序時，優化版本只需 n-1 次比較

空間複雜度

O(1)

只需常數空間用於臨時變數，In-place 排序

詳細分析

時間複雜度推導：

第一次遍歷需要 (n-1) 次比較

第二次遍歷需要 (n-2) 次比較

...

第 (n-1) 次遍歷需要 1 次比較

總共：(n-1) + (n-2) + ... + 1 = n(n-1)/2 ≈ n²

⚖️ 優點與缺點

✓ 優點

  • 實現簡單易懂
  • 空間複雜度低 (O(1))
  • 穩定排序
  • 對小規模陣列有效
  • 對接近排序的陣列高效
  • 不需要額外的記憶體

✗ 缺點

  • 時間複雜度高 (O(n²))
  • 對大規模陣列不適用
  • 效率遠低於其他演算法
  • 實務中很少使用
  • 適應性差（無法利用陣列結構）
  • 內層迴圈密集

🎯 適用場景

何時使用氣泡排序？

  • 教育用途： 學習排序演算法基礎
  • 小規模數據： 少於 50 個元素的陣列
  • 已排序數據： 陣列幾乎已排序時
  • 空間限制： 需要 O(1) 額外空間的場景

推薦替代方案

  • 中等規模 (50-1000)： 插入排序或選擇排序
  • 大規模 (1000+)： 快速排序、合併排序、堆排序
  • 通用場景： 內建的 sort() 函數（通常使用 Timsort 等混合排序）
