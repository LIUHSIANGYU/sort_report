📌 插入排序法 (Insertion Sort)

視覺化演示與複雜度分析

📖 操作原理

核心概念

插入排序法就像整理撲克牌一樣。將陣列分為已排序和未排序兩部分。每次從未排序部分取出一個元素，比較後插入到已排序部分的正確位置。透過逐步插入，最終整個陣列完成排序。

插入過程

從陣列第二個元素開始，依次與已排序部分進行比較。找到恰當的位置後，將該元素插入並將已排序部分後移。不斷重複此過程，直到所有元素都被插入到已排序列中。

📖 演算法說明

插入排序法的原理

插入排序透過逐步建立有序序列的方式進行排序。它將陣列分為已排序和未排序兩部分，每次從未排序部分取出一個元素，將其插入到已排序部分的正確位置。

演算法偽代碼：

INSERTION-SORT(A)
for j = 2 to A.length
key = A[j]
i = j - 1
while i > 0 and A[i] > key
A[i+1] = A[i]
i = i - 1
A[i+1] = key
⏱️ 時間與空間複雜度

最壞情況

O(n²)

陣列完全反序排列時需要最多的比較

平均情況

O(n²)

隨機排列的陣列需要約 n²/4 次比較

最好情況

O(n)

陣列已排序時，只需 n-1 次比較

空間複雜度

O(1)

只需常數空間，In-place 排序

⚖️ 優點與缺點

✓ 優點

  • 實現簡單易懂
  • 空間複雜度低 (O(1))
  • 穩定排序
  • 對小規模或幾乎排序的陣列非常高效
  • 線上排序（邊讀入邊排序）

✗ 缺點

  • 時間複雜度高 (O(n²))
  • 對大規模陣列不適用
  • 移動操作較多
  • 效率遠低於快速排序等高級演算法
