快速排序法的原理

快速排序是一種分散-征服演算法，它通過選擇一個樞紐元素（pivot），將陣列分為兩部分：小於樞紐的元素和大於樞紐的元素。然後對這兩部分遞迴地進行相同的過程，直到整個陣列排序完成。

基本步驟：

  1. 從陣列中選擇一個樞紐元素
  2. 使用分割操作將陣列分為兩部分
  3. 所有小於樞紐的元素移到左邊，大於樞紐的移到右邊
  4. 遞迴地對左右兩個子陣列進行上述操作
  5. 當子陣列大小為 1 時，陣列排序完成

JavaScript 實現：

async function quickSort(arr, low = 0, high = arr.length - 1) { if (low < high) { const pi = await partition(arr, low, high); await quickSort(arr, low, pi - 1); await quickSort(arr, pi + 1, high); } } async function partition(arr, low, high) { const pivot = arr[high]; let i = low - 1; for (let j = low; j < high; j++) { comparisons++; await highlightBars([j, high], 'comparing'); if (arr[j] < pivot) { i++; [arr[i], arr[j]] = [arr[j], arr[i]]; moves++; updateVisualization(); } } [arr[i + 1], arr[high]] = [arr[high], arr[i + 1]]; moves++; updateVisualization(); return i + 1; }
⏱️ 時間與空間複雜度

最壞情況

O(n²)

當樞紐始終为最小或最大元素時，每次只分割出1個元素

平均情況

O(n log n)

樞紐較為平衡時，陣列平均被分成兩個相等的部分

最好情況

O(n log n)

樞紐為中位數，陣列完全均勻分割

空間複雜度

O(log n)

遞迴調用棧的深度為 O(log n) 的平均情況

⚖️ 優點與缺點

✓ 優點

  • 平均時間複雜度好 (O(n log n))
  • In-place 排序，空間效率高
  • 在實踐中性能非常好
  • 可以利用快取實現優化
  • 適用於大規模數據集

✗ 缺點

  • 最壞情況時間複雜度為 O(n²)
  • 不穩定排序
  • 遞迴實現需要額外棧空間
  • 樞紐選擇不好會嚴重影響效率
  • 對小規模數據集可能不如簡單排序高效
