🔀 合併排序法 (Merge Sort)

視覺化演示與複雜度分析

📖 操作原理

核心概念

合併排序法採用分治策略。先將陣列遞迴分成更小的子陣列，直到每個子陣列只含一個元素（已排序）。然後開始合併，將兩個有序的子陣列合併成一個更大的有序陣列。重複此過程直到整個陣列合併完成。

分治過程

分割階段不斷將陣列二分，直到每個元素獨立。合併階段比較兩個有序子陣列的元素，按順序放入新陣列。透過逐層向上合併，最終形成完全有序的陣列。

📖 演算法說明

合併排序法的原理

合併排序是一種分治演算法，採用「分而治之」的策略。它將陣列分割成更小的子陣列，然後遞迴地對每個子陣列進行排序，最後將已排序的子陣列合併回一個完整的排序陣列。

演算法偽代碼：

MERGE-SORT(A, p, r)
if p < r
q = floor((p + r) / 2)
MERGE-SORT(A, p, q)
MERGE-SORT(A, q+1, r)
MERGE(A, p, q, r)

MERGE(A, p, q, r)
n1 = q - p + 1
n2 = r - q
create L[1..n1+1] and R[1..n2+1]
for i = 1 to n1
L[i] = A[p + i - 1]
for j = 1 to n2
R[j] = A[q + j]
L[n1+1] = ∞
R[n2+1] = ∞
i = 1
j = 1
for k = p to r
if L[i] ≤ R[j]
A[k] = L[i]
i = i + 1
else
A[k] = R[j]
j = j + 1
⏱️ 時間與空間複雜度

最壞情況

O(n log n)

無論陣列如何排列，複雜度始終是 n log n

平均情況

O(n log n)

隨機排列的陣列也需要 n log n 次操作

最好情況

O(n log n)

即便陣列已排序，仍需 n log n 次操作

空間複雜度

O(n)

需要額外空間存儲臨時陣列

⚖️ 優點與缺點

✓ 優點

  • 穩定的複雜度 O(n log n)
  • 適用於大規模陣列排序
  • 穩定排序
  • 適合外部排序和並行處理
  • 較小元素集合中表現出色

✗ 缺點

  • 需要額外 O(n) 空間
  • 對已排序陣列無優化
  • 遞迴調用開銷
  • 小規模資料下不如簡單排序高效
