選擇排序法的原理

選擇排序是一種簡單的排序演算法。它每次從未排序的元素中找到最小的元素，放在已排序部分的末尾。通過這樣的選擇過程，逐步建立排序後的陣列。

基本步驟：

  1. 在未排序的元素中找到最小值
  2. 將最小值與未排序部分的第一個元素交換
  3. 移動已排序部分的邊界向右移動一位
  4. 重複此過程，直到整個陣列排序完成

JavaScript 實現：

function selectionSort(arr) { const n = arr.length; let comparisons = 0; let swaps = 0; for (let i = 0; i < n - 1; i++) { let minIndex = i; // 在未排序部分找最小值 for (let j = i + 1; j < n; j++) { comparisons++; if (arr[j] < arr[minIndex]) { minIndex = j; } } // 若最小值不在目前位置，進行交換 if (minIndex !== i) { [arr[i], arr[minIndex]] = [arr[minIndex], arr[i]]; swaps++; } } return { comparisons, swaps }; }
⏱️ 時間與空間複雜度

最壞情況

O(n²)

需要進行所有的比較和交換操作

平均情況

O(n²)

隨機排列的陣列需要約 n²/2 次比較

最好情況

O(n²)

即使陣列已排序，仍需所有比較操作

空間複雜度

O(1)

只需常數空間用於臨時變數，In-place 排序

詳細分析

時間複雜度推導：

第一次遍歷需要 (n-1) 次比較

第二次遍歷需要 (n-2) 次比較

...

第 (n-1) 次遍歷需要 1 次比較

總共：(n-1) + (n-2) + ... + 1 = n(n-1)/2 ≈ n²

關鍵區別：選擇排序的比較次數是固定的，無論輸入如何排列都是 n²/2 次比較，但交換次數最多 n-1 次。

⚖️ 優點與缺點

✓ 優點

  • 實現簡單易懂
  • 空間複雜度低 (O(1))
  • 交換次數少 (最多 n-1 次)
  • 對小規模陣列有效
  • 不適應性排序但穩定性可控
  • 不需要額外的記憶體

✗ 缺點

  • 時間複雜度高 (O(n²))
  • 比較次數固定，無法優化
  • 對大規模陣列不適用
  • 效率遠低於其他演算法
  • 實務中很少使用
  • 無法利用已排序的資料優勢
🎯 適用場景

何時使用選擇排序？

  • 教育用途： 學習排序演算法基礎
  • 小規模數據： 少於 50 個元素的陣列
  • 交換成本高： 當交換操作的成本遠高於比較時
  • 空間限制： 需要 O(1) 額外空間的場景

與氣泡排序的比較

  • 相似點： 時間複雜度都是 O(n²)，空間複雜度都是 O(1)
  • 差異點： 選擇排序的比較次數固定，氣泡排序對已排序數據有優勢
  • 推薦： 通常氣泡排序因易實現而更常用，但選擇排序交換更少

推薦替代方案

  • 中等規模 (50-1000)： 快速排序或插入排序
  • 大規模 (1000+)： 快速排序、合併排序、堆排序
  • 通用場景： 內建的 sort() 函數（通常使用 Timsort 等混合排序）
